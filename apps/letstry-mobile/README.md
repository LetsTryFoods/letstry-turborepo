# letstry-mobile

The new Expo Router mobile application for LetsTry Foods, built with a feature-based, scalable architecture.

---

## 🚀 Quick Start

```bash
# From the monorepo root

pnpm mobile:start       # Start Expo dev server
pnpm mobile:android     # Launch on Android emulator
pnpm mobile:ios         # Launch on iOS simulator
pnpm mobile:ngrok       # Start with ngrok (for physical device testing)
```

```
│
├── app/                         # 📍 SCREENS & NAVIGATION (Expo Router)
│   │                            #    File names = URL routes. No manual navigation setup.
│   ├── _layout.tsx              #    Root layout: wraps everything in Apollo, TanStack Query, SafeArea providers
│   ├── (auth)/                  #    Auth group: screens only shown to unauthenticated users
│   │   ├── _layout.tsx
│   │   └── login.tsx            #    /login route
│   └── (tabs)/                  #    Main app tab group: screens for logged-in users
│       ├── _layout.tsx          #    Bottom tab bar configuration
│       ├── index.tsx            #    / (Home tab)
│       └── profile.tsx          #    /profile tab
│
├── src/                         # 📦 ALL SOURCE CODE (non-screen)
│   │
│   ├── features/                # 🧩 FEATURE MODULES — the heart of the app
│   │   │                        #    Each feature is fully self-contained.
│   │   └── auth/                #    Example: the Auth feature
│   │       ├── components/      #    UI components SPECIFIC to this feature (e.g., LoginForm.tsx)
│   │       ├── hooks/           #    TanStack Query hooks (e.g., useLogin.ts) — data-fetching logic
│   │       ├── schemas/         #    Zod validation schemas for forms (e.g., auth.schema.ts)
│   │       ├── services/        #    Raw API calls — calls Apollo/axios and returns typed data
│   │       └── types.ts         #    TypeScript interfaces for this feature's data models
│   │
│   ├── lib/                     # 🔧 SHARED INFRASTRUCTURE — used by all features
│   │   ├── apollo-client.ts     #    Apollo Client setup: auth headers, error handling, network logger
│   │   ├── query-client.ts      #    TanStack QueryClient singleton with default options
│   │   └── graphql/             #    GraphQL query/mutation documents (raw gql tags)
│   │       └── auth.ts          #    LOGIN_MUTATION, GET_ME_QUERY, etc.
│   │
│   ├── components/              # 🎨 GLOBAL REUSABLE UI COMPONENTS
│   │   ├── ui/                  #    Primitive components used across all features
│   │   │   ├── Button.tsx       #    Button with primary/secondary/outline variants + loading state
│   │   │   └── Input.tsx        #    Text input with label, error message, and consistent styling
│   │   └── layout/              #    Layout wrappers (ScreenWrapper, Header, etc.)
│   │
│   ├── styles/                  # 🎨 DESIGN TOKENS — single source of truth
│   │   └── theme.ts             #    All colors, spacing, border radii, font sizes, font weights
│   │
│   └── gql/                     # ⚙️  AUTO-GENERATED — do not edit manually
│       └── graphql.ts           #    TypeScript types generated by graphql-codegen from the backend schema
│
├── scripts/
│   └── start-with-ngrok.sh     # 🚇 Auto-starts ngrok and passes the URL to Expo
│
├── .env                         # 🔑 Environment variables (not committed to git)
│                                #    EXPO_PUBLIC_GRAPHQL_URL=http://localhost:3001/graphql
├── codegen.ts                   # ⚙️  graphql-codegen config — run with `pnpm codegen`
└── app.json                     # ⚙️  Expo app config (name, bundle ID, plugins)
```

## 🧩 Adding a New Feature

1. Create a folder under `src/features/your-feature/`
2. Add the sub-folders: `components/`, `hooks/`, `schemas/`, `services/`, `types.ts`
3. Write GraphQL docs in `src/lib/graphql/your-feature.ts`
4. Create a screen in `app/(tabs)/your-screen.tsx` — import from the feature, no logic in screens

**Rule:** Screens only render components. Components only render UI. Logic lives in hooks. Data lives in services.

## 🌐 ngrok (Physical Device Testing)

ngrok creates a public HTTPS tunnel to your local backend so a physical device can reach it.

```bash
# From the monorepo root — one command does everything:
pnpm mobile:ngrok
```

This runs `scripts/start-with-ngrok.sh`, which:
1. Starts ngrok on port `3001` (set `BACKEND_PORT=XXXX` to override)
2. Automatically reads the public URL from ngrok's local API
3. Launches Expo with `EXPO_PUBLIC_GRAPHQL_URL` set to the ngrok URL

## 🔁 Codegen (Keeping Types in Sync)

After changing any GraphQL query/mutation, regenerate types:

```bash
pnpm --filter letstry-mobile codegen
```

This reads from `codegen.ts` and writes to `src/gql/graphql.ts`. **Never edit `graphql.ts` manually.**
